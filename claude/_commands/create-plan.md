## 概要

コンテキストから実装計画を立案し、チェックリスト形式で/docs/plan配下に保存する計画作成コマンド

これまでのコンテキストの内容を元に実装計画を立ててください。

## 実行フロー

1. **コンテキストの分析**
   - ユーザーの要求内容を理解する
   - 既存のコードベースを確認し、影響範囲を特定する
   - 必要な機能や変更点を整理する

2. **仕様の確認**
   - 仕様が不足している場合は、実装計画を作成せず質問を返す
   - 不明点や曖昧な部分を明確にする

3. **計画のブレイクダウン**
   - 実装をフェーズ単位に分解する
   - 各フェーズ内のタスクを細分化する
   - チェックリスト形式（`- [ ]`）で記述する

4. **計画書の作成**
   - `/docs/plan`配下にMarkdownファイルとして保存
   - ファイル名は実装内容が明確に伝わる名前にする（例: `feature-authentication-plan.md`）

5. **計画の確認**
   - 作成した計画をユーザーに提示し、確認を得る

## 注意事項

- **可能な限りブレイクダウンすること**
- **既存のコードベースを可能な限り維持し、変更範囲を最小限にすること**
- **仕様が不足している場合は、コードを書かずに質問を返すこと**
- **実装事項の進捗が一目瞭然なようにチェックリスト式フォーマットにすること**
- **ファイル名は実装内容が明確に伝わる名前にすること**

## 使用例

```bash
# コンテキストから実装計画を作成
/create-plan

# 特定の機能の実装計画を作成
/create-plan "ユーザー認証機能の実装"
```

## 期待される動作

- コンテキストを分析し、実装に必要な要素を整理
- 仕様不足がある場合は質問を返す
- フェーズ単位でブレイクダウンされたチェックリスト形式の計画書を作成
- `/docs/plan`配下に分かりやすいファイル名で保存
- 作成した計画書をユーザーに提示
