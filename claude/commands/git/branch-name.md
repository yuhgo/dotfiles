## 概要

ユーザーのプロンプトからgitブランチ名候補を生成し、選択、切り替えを行う

# Gitブランチ名生成・選択・切り替えコマンド

このコマンドは、ユーザーから受け取ったプロンプトを元にgitブランチ名の候補を生成し、選択されたブランチに切り替えるためのコマンドです。

## 実行フロー

1. **ブランチ名候補生成**: ユーザーのプロンプトから10件のブランチ名候補を生成
2. **候補表示**: 番号付きリスト形式で候補を表示し、各候補に解説を追加
3. **ユーザー選択**: どの候補を採用するかユーザーに質問
4. **ブランチ切り替え**: 選択されたブランチ名でgit checkoutコマンドを実行

## ブランチ名生成ルール

### 命名規則
- `feature/`: 新機能追加
- `fix/`: バグ修正
- `refactor/`: リファクタリング
- `docs/`: ドキュメント更新
- `test/`: テスト追加・修正
- `chore/`: 雑務・設定変更
- `perf/`: パフォーマンス改善
- `style/`: スタイル変更

### 命名ガイドライン
- 英語のケバブケース（小文字・ハイフン区切り）
- 内容を端的に表現（30文字以内推奨）
- 動詞始まりで具体的な行動を表現
- 略語は避け、理解しやすい単語を使用

## 出力フォーマット

```
## ブランチ名候補

1. feature/update-package-screen-titles
   解説: 荷物画面のタイトル更新というメイン機能を直接的に表現。シンプルで分かりやすい命名。

2. fix/login-validation-error
   解説: ログイン時のバリデーションエラー修正。問題領域と修正内容が明確。

...（10件まで）

## 選択してください
何番のブランチ名を採用しますか？（1-10の数字で回答）
```

## 実行例

### ユーザー入力例
```
荷物画面のタイトルを更新したい
```

### システム応答例
```
## ブランチ名候補

1. feature/update-package-screen-titles
   解説: 荷物画面のタイトル更新というメイン機能を直接的に表現。シンプルで分かりやすい命名。

2. feature/modify-shipment-page-headers
   解説: 荷物ページのヘッダー修正に焦点。より技術的な表現でUI変更であることが明確。

3. feature/refine-package-display-titles
   解説: パッケージ表示タイトルの改良。継続的改善のニュアンスを含む。

4. feature/enhance-cargo-screen-labels
   解説: 荷物画面ラベルの拡張。機能強化の側面を強調。

5. feature/redesign-shipment-title-section
   解説: 荷物タイトルセクションの再設計。大きめの変更を想定した命名。

6. feature/update-delivery-package-names
   解説: 配送パッケージ名の更新。配送コンテキストを明示。

7. feature/improve-package-header-text
   解説: パッケージヘッダーテキストの改善。文言改善に特化した表現。

8. feature/modify-cargo-display-strings
   解説: 荷物表示文字列の修正。文字列変更の技術的側面を強調。

9. chore/update-package-screen-copy
   解説: パッケージ画面コピーの更新。文言変更を雑務として分類。

10. style/adjust-shipment-title-styling
    解説: 荷物タイトルスタイリング調整。見た目の調整に焦点。

## 選択してください
何番のブランチ名を採用しますか？（1-10の数字で回答）
```

### ユーザー選択例
```
1
```

### システム最終応答例
```
選択されたブランチ名でgit checkoutコマンドを実行します：

`git checkout -b feature/update-package-screen-titles`

このコマンドを実行してよろしいですか？（y/n）
```

### nと回答された場合の処理
ユーザーが「n」と回答した場合は、`branch-command.md` ファイルを作成し、以下の内容をまとめる：

1. **ファイル最上部**: 選択されたgit checkoutコマンドを記載
2. **タスク内容**: 今回依頼されたタスクの詳細
3. **採用ブランチ名**: ユーザーが選択したブランチ名とその理由
4. **実行されなかった理由**: なぜコマンドが実行されなかったかの記録

#### branch-command.mdのフォーマット
```markdown
# Git Checkout コマンド
git checkout -b [選択されたブランチ名]

## タスク内容
[ユーザーから依頼されたタスクの詳細]

## 採用されたブランチ名
**ブランチ名**: [選択されたブランチ名]
**選択理由**: [該当する候補の解説]
**選択番号**: [ユーザーが選択した番号]

## 実行状況
**実行日時**: [年月日 時分]
**実行ステータス**: 未実行（ユーザーが「n」を選択）
**備考**: [追加の備考があれば記載]

## 生成された全候補
[生成された10件のブランチ名候補リスト]
```

## 注意事項

- ブランチ名は既存のgitリポジトリの命名規則に従って生成してください
- 日本語プロンプトから英語ブランチ名への適切な翻訳を行ってください
- 生成されたブランチ名が既存ブランチと重複していないか確認してください
- git checkoutコマンド実行前に確認を取ってください
- 「n」回答時のbranch-command.mdファイル作成を忘れずに実行してください
