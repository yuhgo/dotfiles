# Gitブランチ名生成・選択・切り替えスキル

ユーザーから受け取ったプロンプトを元にgitブランチ名の候補を**3件に厳選**して提示し、選択されたブランチに切り替える。

## 実行フロー

1. **ブランチ名候補生成**: ユーザーのプロンプトから最適なブランチ名を3件厳選する
2. **AskUserQuestionで選択**: 3つの候補をAskUserQuestionツールで提示し、ユーザーに選んでもらう
3. **ブランチ切り替え**: 選択されたブランチ名で `git checkout -b` を実行

## ブランチ名生成ルール

### Prefix
- `feature/`: 新機能追加
- `fix/`: バグ修正
- `refactor/`: リファクタリング
- `docs/`: ドキュメント更新
- `test/`: テスト追加・修正
- `chore/`: 雑務・設定変更
- `perf/`: パフォーマンス改善
- `style/`: スタイル変更

### 命名ガイドライン
- 英語のケバブケース（小文字・ハイフン区切り）
- 内容を端的に表現（30文字以内推奨）
- 動詞始まりで具体的な行動を表現
- 略語は避け、理解しやすい単語を使用

## 候補選択の実装

AskUserQuestionツールを使って、以下の形式で3つの候補を提示する：

```
AskUserQuestion({
  questions: [{
    question: "どのブランチ名を使いますか？",
    header: "ブランチ名",
    options: [
      { label: "feature/add-user-auth", description: "ユーザー認証機能の追加を直接的に表現" },
      { label: "feature/implement-login-flow", description: "ログインフロー実装に焦点を当てた表現" },
      { label: "feat/setup-auth-middleware", description: "認証ミドルウェアのセットアップという技術的な表現" }
    ],
    multiSelect: false
  }]
})
```

## 3件厳選のポイント

10件ではなく3件に絞る理由は、選択の負担を減らすため。以下の観点で厳選する：

1. **最もシンプルで直接的な表現**: タスクの本質を端的に表すもの
2. **技術的な観点の表現**: 実装内容に焦点を当てたもの
3. **スコープが異なる表現**: 異なる粒度やアプローチのもの

## 選択後の処理

ユーザーが候補を選択したら：

1. 既存ブランチと重複していないか `git branch --list` で確認
2. `git checkout -b <選択されたブランチ名>` を実行
3. 実行結果を報告

### 「Other」が選択された場合

ユーザーがカスタム入力した場合は、その内容をブランチ名として使用する。

## 注意事項

- 日本語プロンプトから英語ブランチ名への適切な翻訳を行う
- 既存ブランチとの重複チェックを実行前に行う
- git checkout実行前に必ずユーザーの選択を待つ
