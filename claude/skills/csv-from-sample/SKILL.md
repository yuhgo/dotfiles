# CSV作成スキル（サンプルデータから）

ユーザーが提供するCSVサンプルデータ（テーブル形式やTSV形式）を解析し、会計ソフト用のCSVファイルを作成する。

## 前提条件

- 出力先プロジェクト: 確定申告用のリポジトリ（`csv/` フォルダが存在すること）
- エンコーディング: **Shift_JIS**（会計ソフト互換）
- フォーマット: **全フィールドをダブルクォートで囲む**
- 金額のカンマ区切りは除去して数値のみにする（例: `660,094` → `660094`）

## CSVヘッダー（固定19列）

```
"取引No","取引日","借方勘定科目","借方補助科目","借方部門","借方取引先","借方税区分","借方インボイス","借方金額(円)","貸方勘定科目","貸方補助科目","貸方部門","貸方取引先","貸方税区分","貸方インボイス","貸方金額(円)","摘要","タグ","メモ"
```

## 実行フロー

### Step 1: サンプルデータの解析

ユーザーが提供するサンプルデータから以下を読み取る:

1. **取引先名/カテゴリ名** — フォルダ名・ファイル名に使用
2. **CSVの種類** — 売上、入金、経費、開始残高など
3. **各行のデータ** — 19列に対応するフィールドの値
4. **複数行セットの有無** — 入金のように1取引が複数行（普通預金+支払手数料など）で構成されるパターンか確認

### Step 2: ユーザーへの確認

以下をユーザーに確認する（サンプルデータから明確に読み取れる場合はスキップ可）:

- ファイル名（例: `売上_本田運送.csv`, `入金_本田運送.csv`）
- 配置フォルダ（例: `csv/本田運送/`）
- サンプルデータの年度（提供された年度をそのまま使うか変更するか）

### Step 3: CSVファイルの生成

Python3で以下の処理を行う:

```python
import csv
import os

base_dir = "<プロジェクトルート>/csv"
target_dir = os.path.join(base_dir, "<取引先名>")
os.makedirs(target_dir, exist_ok=True)

header = [
    "取引No","取引日","借方勘定科目","借方補助科目","借方部門",
    "借方取引先","借方税区分","借方インボイス","借方金額(円)",
    "貸方勘定科目","貸方補助科目","貸方部門","貸方取引先",
    "貸方税区分","貸方インボイス","貸方金額(円)","摘要","タグ","メモ"
]

data = [
    # ユーザーのサンプルデータをここに展開
]

output_path = os.path.join(target_dir, "<ファイル名>.csv")
with open(output_path, "w", encoding="shift_jis", newline="") as f:
    writer = csv.writer(f, quoting=csv.QUOTE_ALL)
    writer.writerow(header)
    writer.writerows(data)
```

### Step 4: 出力確認

生成したCSVファイルを `iconv -f Shift_JIS -t UTF-8` で表示し、ユーザーに内容を確認してもらう。

## データ変換ルール

| 入力 | 出力 | 例 |
|------|------|-----|
| カンマ付き金額 | カンマ除去 | `660,094` → `660094` |
| 空セル | 空文字列 `""` | |
| 年度が前年のサンプル | 当年に変更（確認する） | `2024/` → `2025/` |
| タブ区切りの入力 | CSVカンマ区切りに変換 | |

## フォルダ構成ルール

```
csv/
├── <取引先名>/
│   ├── 売上_<取引先名>.csv
│   ├── 入金_<取引先名>.csv
│   └── ...
├── <勘定科目名>/
│   ├── 借方_<勘定科目名>.csv
│   └── 貸方_<勘定科目名>.csv
└── ...
```

- 取引先ごとのCSVは取引先名のフォルダにまとめる
- 勘定科目ごとの汎用CSVは勘定科目名のフォルダにまとめる

## ファイル命名規則

- `<種類>_<取引先名>.csv`
- 種類の例: `売上`, `入金`, `経費`, `借方`, `貸方`

## エラーハンドリング

| ケース | 対応 |
|--------|------|
| サンプルの列数が19列と一致しない | 不足列を空文字で補完、ユーザーに報告 |
| フォルダが既に存在する | そのまま使用（上書きしない、追加） |
| 同名ファイルが既に存在する | ユーザーに上書きするか確認 |
| 金額が数値でない | ユーザーに確認 |
