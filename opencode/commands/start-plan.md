## 概要

実装計画書に沿って実装を進め、フェーズごとにチェックリストを完了（✅）にする実行コマンド

実装計画書（`/docs/plan`配下のMarkdownファイル）に沿って、実装を進めてください。

## 実行フロー

1. **計画書の読み込み**
   - ユーザーから指定されたplan.mdファイルを参照する
   - 指定されていない場合は、ユーザーに計画書のパスを確認する

2. **現在の進捗確認**
   - 計画書のチェックリストを確認し、未完了のフェーズ・タスクを特定する
   - 既に完了しているタスク（✅）はスキップする

3. **フェーズ単位での実装**
   - 未完了のフェーズから順番に実装を進める
   - 各フェーズ内のタスクを1つずつ実装する
   - タスク完了後、計画書のチェックリストを更新（`- [ ]` → `- [x]` または ✅ に変更）

4. **進捗の可視化**
   - TodoWriteを活用して、現在実行中のフェーズとタスクを明確にする
   - フェーズ完了時には、ユーザーに完了報告を行う

5. **検証とレビュー**
   - 各フェーズ完了後、動作確認やテストを実施する
   - 問題がある場合は、計画書に記録して修正する

6. **全フェーズ完了**
   - すべてのフェーズが完了したら、最終確認を行う
   - 計画書を最終状態に更新し、完了報告を行う

## 注意事項

- **既存のコードベースを可能な限り維持する**
- **変更範囲を最小限にする**
- **仕様が不明な場合は、実装を止めて質問する**
- **フェーズごとに進捗を更新し、可視化する**
- **テストやバリデーションを忘れない**

## 使用例

```bash
# 実装計画書に沿って実装を開始
/start-plan

# 特定の計画書を指定して実行
/start-plan feature-authentication-plan.md
```

## 期待される動作

- 計画書を読み込み、未完了タスクを特定
- フェーズごとに実装を進め、完了したらチェックリストを更新（✅）
- 進捗をTodoWriteで可視化
- 全フェーズ完了後、最終確認と完了報告
